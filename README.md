# GlobalNetSpeedConfigs (GNSC)

**GlobalNetSpeedConfigs（简称 GNSC）** 是一套免费、开源的"自动为特定网站挑选最快 IP 并写入本机 hosts"工具。它在 **不使用代理 / VPN** 的前提下，帮你把系统对特定网站（当前主要是 **亚马逊 Amazon 相关域名**）的访问，就近指向"实测延迟最低"的服务器 IP，从而改善加载速度与稳定性。一条命令安装，之后每天全自动更新，无需手动维护 hosts。

## 一、这是什么？能解决什么问题

很多人访问亚马逊等站点时会遇到图片/页面加载慢、卡顿、时断时续的问题，这往往是因为本机被解析到了一台"较远或较慢"的服务器。GNSC 通过维护一份优化过的 hosts，把域名直接钉到"更快的 IP"上来缓解这些问题。它由两部分协作完成：

- **服务端（数据生产）：** 一台部署在河南联通宽带上的飞牛 NAS，每天定时测速，产出各地区/运营商对应的 `hosts_*.txt` 配置并推送到本仓库。
- **客户端（本项目脚本 `gnsc.ps1`）：** Windows 一键安装，注册为每天自动运行的计划任务，自动识别你所在的地区/运营商，下载最匹配的配置并安全写入本机 hosts。

**适合谁：** ① 访问亚马逊等站点慢、卡、不稳定的用户；② 不想折腾代理、只想"就近连到更快 IP"的用户；③ 希望一次安装、之后全自动、无需每天手动改 hosts 的用户。

### 它的原理，以及为什么安全

亚马逊等大型网站普遍采用 **CDN（内容分发网络）** 技术：同一个域名在各地部署了大量服务器节点，正常情况下由 DNS 就近调度。但 **DNS 返回的节点未必是"对你当前网络而言延迟最低"的那个**——它常受运营商本地缓存、解析调度策略、跨网互联质量等影响，把你导向一个较远或较慢的节点，于是出现加载慢、卡顿。GNSC 的做法是：在服务端针对"你所在地区 + 运营商"实测出最快的节点 IP，再用本机 hosts 精确指定，把访问 **固定到最优线路** 上，相当于替代了 DNS 里"不够聪明"的那一步。

**它不会导致卖家担心的"账户关联"。** 这种方式只是优化你本机到网站服务器之间的 **网络连接路径**：不使用代理 / VPN，不改变你的真实公网出口 IP，也不共享浏览器指纹或登录环境。网站看到的仍是你原本真实的本地网络出口，只是连到了同一网站在 CDN 上"更快的那台服务器"。因此它与"多账户共用同一网络/代理环境"导致的账户关联是两回事，不会因为使用本工具而产生关联风险。

> ⚠️ **适用范围（重要）：** 当前项目仅提供 **中国 · 河南 · 联通（`hosts_CN_HN_CU_Amazon.txt`）** 这一份配置文件，因此 **目前只适合"河南联通"网络环境的用户**。其它省份、其它运营商（电信 / 移动等）暂无对应配置，即使安装，客户端也会因匹配不到配置文件而不生效。后续会视情况逐步补充更多地区与运营商的配置。

## 二、工作原理

### 服务端（飞牛 NAS · 河南联通）

每天凌晨定时运行：先测试目标域名（主要是亚马逊相关域名）在多个区域 DNS 返回的服务器 IP（通过伪造子域实现），再用 NAS 本机对每个候选 IP 做实测，挑出每个域名延迟最低的 IP，生成含"域名 → IP"对应关系的 `hosts_*.txt`，并推送到本仓库根目录。文件顶部带 `# GENERATED_TIME:` 时间戳，供客户端判断新旧与过期。

### 客户端（gnsc.ps1）

- **安装时：** 自动 UAC 提权 → 从多镜像择优下载脚本到 `C:\ProgramData\GlobalNetSpeedConfigs\` → 注册以 **SYSTEM** 运行的计划任务（每天 04:00 + 开机触发，重启免登录、无感运行）→ 立即同步一次。
- **每天运行时：** 查询你的 **公网出口 IP** 的国家 / 省份 / 运营商（经 `ipinfo.io`）→ 拼出配置名（如 `hosts_CN_HN_CU_Amazon.txt`）→ 下载 → 用 `BEGIN/END` 标记块把内容安全写入本机 hosts → 刷新 DNS 缓存。整个过程只改自己标记块内的内容，不影响你已有的其它 hosts 记录。

## 三、快速开始（Windows）

在 **cmd** 或 **PowerShell** 中粘贴下面这一行并回车即可（脚本会自动请求管理员权限，无需手动"以管理员身份运行"）：

```powershell
powershell -c "irm gh-proxy.com/https://raw.githubusercontent.com/26eg/GlobalNetSpeedConfigs/main/gnsc.ps1|iex"
```

- **未安装：** 提示"回车立即安装，其它键退出"。
- **已安装：** 显示本地 / 在线版本，并给出菜单——`[1]` 重装 / 升级（覆盖安装）、`[2]` 卸载、其它键退出。
- **系统要求：** Windows 10 / 11，内置的 Windows PowerShell 5.1 及以上，安装需管理员权限。

## 四、配置文件命名规则

配置文件按"国家_省份_运营商_业务"命名，格式为 `hosts_{国家}_{省份}_{运营商}_{业务}.txt`，例如 **`hosts_CN_HN_CU_Amazon.txt`**。客户端会依据你的公网出口 IP 自动匹配对应文件。常见编码对照：

| 字段 | 示例编码 | 含义 |
| --- | --- | --- |
| 国家 | `CN` | 中国 |
| 省份 | `HN` | 河南 |
| 运营商 | `CU / CT / CM` | 中国联通 / 中国电信 / 中国移动 |
| 业务 | `Amazon` | 亚马逊相关域名 |

## 五、安装后包含哪些内容

- **安装目录 `C:\ProgramData\GlobalNetSpeedConfigs\`：** `gnsc.ps1`（主脚本）、`worker.log`（运行日志）、`hosts.*.bak`（首次修改前的原始 hosts 备份）。
- **计划任务 `GlobalNetSpeedConfigs`：** 以 SYSTEM 运行，每天 04:00 及开机时触发。
- **本机 hosts：** 新增一段由 `# >>> GlobalNetSpeedConfigs BEGIN >>>` 与 `# <<< GlobalNetSpeedConfigs END <<<` 包裹的内容块。

## 六、更新与自愈机制

- **按需更新：** 每天比对在线配置的时间戳，只有更新时才写入 hosts，避免无谓改动。
- **脚本自更新：** 后台任务运行时若发现在线版本更高，会静默替换本机脚本，用户无需手动重装。
- **过期保护：** 若连续 14 天拿不到在线配置，自动移除本地内容块，防止过期 IP 导致访问异常。
- **安全写入：** 采用"临时文件 + 原子替换 + 最多 6 次重试"，即便被杀软占用也 **绝不会把 hosts 清成 0KB**；首次修改前自动备份原始 hosts。

## 七、卸载

重新运行一键安装命令，在菜单中按 `[2]` 即可一键卸载：删除计划任务、清理 hosts 中的 GNSC 内容块、移除安装目录，恢复到安装前状态。

## 八、安全与隐私

- **不采集个人数据：** 仅读取"公网出口 IP 的地理/运营商信息"用于匹配配置，不收集、不上传任何个人隐私。
- **最小化改动：** 只修改自己 `BEGIN/END` 标记内的 hosts 内容，不动你的其它记录；首次修改前自动备份。
- **完全开源：** 脚本与配置均公开，可自行审阅后再使用。
- **杀软提示：** 部分安全软件会拦截对 hosts 的修改，如遇拦截，请将 hosts 文件或 PowerShell 加入信任白名单。

## 九、常见问题（FAQ）

- **装完 hosts 没变化？** 先看 `C:\ProgramData\GlobalNetSpeedConfigs\worker.log`：若显示"无法获取公网地理信息"，多为临时网络问题，可稍后再试或手动在任务计划程序里右键任务"运行"。
- **用 `-File` 直接运行脚本报一堆"意外的标记""缺少右 )"？** 这其实是 **编码问题**，并非语法错误：Windows PowerShell 5.1 在中文系统下会把"无 BOM 的 UTF-8 脚本"按 GBK 误读，导致中文乱码并连带报出一堆假的语法错误。正确用法是直接用本页顶部的一键命令安装/运行（走 `irm | iex` 不受影响）；本项目发布的脚本已采用兼容写法，正常安装即可，无需手动用 `-File` 调用。
- **如何确认已安装成功？** 任务计划程序库中存在 `GlobalNetSpeedConfigs` 任务，且 hosts 中能看到 `# >>> GlobalNetSpeedConfigs BEGIN >>>` 区块即为成功。
- **下载慢 / 连不上 GitHub？** 脚本内置多镜像自动择优（gh-proxy → jsDelivr → 自建域名 → GitHub 原始地址），任一线路不通会自动回退到下一个。
